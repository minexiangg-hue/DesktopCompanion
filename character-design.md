# 🎨 Iconic 角色设计

> **这是最重要的决策之一。** 角色就是产品本身。Labubu、Chiikawa、Pokémon —— 它们能火首先是因为"长得好"。

## 设计原则

1. **一见难忘** — 三笔就能画出来的轮廓，辨识度>细节丰富度
2. **有情绪表达能力** — 角色需要有"脸"，眼睛是关键（参考 Chiikawa 的眼睛即灵魂）
3. **风格兼容** — 不管它模仿特朗普还是鲁迅，角色的视觉形象不能违和
4. **可动可静** — 能卖萌也能高冷，适配不同人格的表现
5. **二维/三维形态一致** — 无论是像素风还是 3D 渲染，角色骨架一致

## 设计方向探索

目前有三个方向，需要定一个走：

---

### 🔵 方向 A：毛绒小怪物系

**参考**：Labubu（龙之子）、Mofusand、Sonny Angel
**关键词**：丑萌、毛茸茸、不对称、捣蛋感
**优势**：
- 辨识度极高（Labubu 的锯齿嘴一眼认出）
- "坏坏的感觉"适合吐槽/搞笑人格
- 可延展性强（穿衣服、挂配件）
**风险**：可能不够"可爱"，部分用户觉得 creepy

> 草图意象：一个小毛球，两只不对称的耳朵，一个狡黠的微笑，圆滚滚的身体，短手短脚。

---

### 🟢 方向 B：治愈大眼睛系

**参考**：Chiikawa（吉伊卡哇）、Miffy、Kuromi
**关键词**：大眼、纯净、治愈、极简
**优势**：
- 普适性强，谁能拒绝 Chiikawa 式大眼睛
- 视觉轻盈，适合桌面长时间存在不厌烦
- 情绪传达直接（眼睛变化 = 心情变化）
**风险**：可能太温和，和"毒舌人格"有反差需处理

> 草图意象：一颗小团子，一双占脸 2/3 的大眼睛，极简的嘴，淡淡的腮红，小小的身体。

---

### 🟡 方向 C：精灵/能量体系

**参考**：Pokémon（皮卡丘/伊布）、Stitch、Totoro
**关键词**：有尾巴、有耳朵、有"生物感"
**优势**：
- 生命力感最强，像真的养了个小东西
- 尾巴/耳朵可以做情绪 indicator（耳朵竖起=兴奋，耷拉=无聊）
- 动作表达更丰富
**风险**：复杂度稍高，设计不好容易像廉价吉祥物

> 草图意象：一个像猫又像狐的小精灵，有大耳朵和蓬松尾巴，肚子是屏幕（显示内容用），眼睛是能量色。

---

## 设计待决策清单

| 问题 | 选项 |
|------|------|
| 角色本体方向 | A 毛绒怪物 / B 大眼睛团子 / C 小精灵 |
| 主色调 | 待定（建议：奶白/淡紫/薄荷绿系，不刺眼） |
| 表情系统 | 像素风点阵 / 简约表情 / 动态表情 |
| 尺寸形态 | 全显示 / 半身 / 仅头部 |
| 是否可换装 | MVP 不做 / MVP 提供 2-3 套 skins / 后期开放 |

## 技术实现路径

```
手绘概念草图 → AI 生成多版本（Midjourney / DALL-E）→ 
选定方向 → SVG 矢量化 → 动画帧设计（呼吸/眨眼/摇摆）→
代码集成（CSS 动画 or Lottie or 精灵图）
```

> **MVP 阶段**可以先使用简洁的 SVG 版本，确保角色在桌面端流畅渲染。角色动画先做三个基本动作：呼吸待机、眨眼、打招呼。
