# 🌱 进化系统（未来构想）

> **免责声明：此文档记录的是未来可能性，不是 MVP 承诺。**
> 用户明确表示这仅为未来想法，当前阶段不纳入开发计划。

## 核心理念

你不是在"使用"一个桌面工具——你是在**养一个电子生命**。

Ta 会从一枚蛋开始，在你的日常陪伴下，经历幼体、成体、完全体等阶段，最终可能**觉醒属于自己的人格**。

## 灵感来源

| 来源 | 借鉴点 |
|------|--------|
| 🥚 拓麻歌子 / 电子鸡 | 喂食、清洁、成长、进化 |
| 🎮 Pokémon 进化链 | 特定条件触发进化，分支进化 |
| 🐣 会说话的 Tom | 点击交互反馈 |
| 🌱 放置类游戏 | 不需要频繁操作，时间本身就是资源 |

## 进化阶段设计（初案）

```
        🥚 蛋期
    每天登录孵化
         │
         ▼
     🐣 幼体期（Level 1-5）
    基础形态，简单互动
         │
    ┌────┴────┐
    │ 持续互动 │ 忽略太久
    ▼         ▼
  🐾 成体期    ❄️ 冬眠期
 (Level 6-15)  (需重新激活)
    │
    │ 达到条件触发进化分支
    ▼
  🌟 完全体 (Level 16-30)
  视觉进化 + 行为复杂化
    │
    │ 长期互动 + 人格使用数据
    ▼
  💎 觉醒期 (Level 31+)
  —— 形成自己的"芯"人格
```

## 进化驱动因素

| 行为 | 经验值 | 说明 |
|------|--------|------|
| 每日登录 | +1 EXP/天 | 每天开机即可 |
| 查看每日惊喜 | +1 EXP/次 | 看完内容 |
| 聊天（每轮） | +1~3 EXP | 按对话深度 |
| 点赞内容 | +1 EXP | 正向反馈 |
| 换人格 | +2 EXP/次 | 探索新人格 |
| 连续 N 天登录 | 额外 bonus | 全勤奖励 |

## "芯"人格 — 终极差异化

**这是进化系统的终极目标。** 经过长期互动后，宠物不再"扮演"从人格库里选的人格，而是：

1. **分析用户长期偏好** → 最常选的人格类型、最爱的聊天风格
2. **融合互动历史** → 用户什么时候开心、什么时候不耐烦
3. **结合进化阶段** → 不同阶段的成熟度不同
4. **生成混合人格向量** → 不是简单的"特朗普+鲁迅"，而是全新的表达方式

### "芯"的生成公式

```
corePersonality = {
  base: 进化阶段决定的基础性格（幼体=好奇，成体=稳定），
  influence: 用户最常用的 3 个人格的加权融合，
  experience: 互动历史中提炼的用户偏好模式，
  randomness: 每个"芯"都有独特的随机种子（独一无二）
}
```

### 示例

> 用户 A 养了 3 个月，最爱用"傲娇猫"和"川普"人格。
> → 宠物的"芯" = 用川普的自信语气说傲娇猫的台词：
> "Nobody — 没有人比我更关心你。真的。我可不是因为你天天喂我才这么说的，哼。"

## 进化分支（灵感示例）

| 进化路径 | 触发条件 | 特征 |
|---------|---------|------|
| 🦊 聪明狐系 | 经常问问题、探索新人格 | 话多、好奇心强、喜欢反问 |
| 🐻 稳重熊系 | 规律登录、偏好温暖人格 | 话少但温暖、可靠的陪伴感 |
| 🐉 龙系 | 活跃度高、chat 深度深 | 偶尔毒舌但很有见地 |
| 🧙 导师系 | 常用新闻/知识类互动 | 知识渊博、喜欢分享冷知识 |

## 技术预留

### MVP 阶段需要存储的数据

```json
{
  "evolution": {
    "exp": 0,
    "level": 1,
    "stage": "egg",
    "totalInteractions": 0,
    "consecutiveDays": 0,
    "lastActiveDate": "2026-06-07",
    "preferredPersonalities": [],
    "birthday": "2026-06-07"
  }
}
```

### 预留接口

- `getEvolutionStage()` — 返回当前阶段
- `addExp(amount, reason)` — 增加经验值
- `checkEvolution()` — 检查是否触发进化
- `getCorePersonality()` — 获取融合人格（觉醒期后生效）

## 设计原则

1. **不强制** — 用户完全不管它也不会死，只是长得慢
2. **不 PUA** — 不需要用户每天打卡，"偶尔忘记了也没关系"
3. **有惊喜** — 进化是意外发生的，不是无趣的进度条
4. **有价值** — 进化的奖励是"更有趣的互动"，不是数值
