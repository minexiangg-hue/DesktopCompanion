# 🏗️ 技术架构

## 技术栈选择

### 候选方案

| 技术栈 | 优点 | 缺点 | 推荐指数 |
|--------|------|------|---------|
| **Electron + React/Svelte** | 生态成熟、跨平台、CSS 动画丰富、开发效率高 | 内存占用偏高 | ⭐⭐⭐⭐ |
| **Tauri（Rust 前端）** | 比 Electron 轻很多、二进制小 | Rust 学习曲线、生态较新 | ⭐⭐⭐⭐ |
| **Python + PyQt / Tkinter** | 与现有技术栈一致、可复用 Avatara/Nuwa | UI 表现力有限，动画吃力 | ⭐⭐⭐ |
| **Flutter Desktop** | 高性能、统一 UI 代码、Hot Reload | 桌面端还在成熟中 | ⭐⭐⭐ |
| **PWA + 系统托盘** | 轻量、无需安装 | 系统集成能力弱 | ⭐⭐ |

> **推荐方案**：Tauri（前端用 Svelte/React） + Python 后端（运行 Avatara/Nuwa 逻辑）。
> 前端负责视觉呈现和动画，Python 后端负责人格调度、内容搜索、用户画像。

## 睡眠/清醒周期

DesktopCompanion 以 **24 小时为周期** 循环。核心就是一句话：**清醒时陪你，睡眠时省钱。**

### 状态机

```
                    ┌──────────────┐
                    │    🟣 睡眠   │
                    │ (20:00-08:00)│
                    └─────┬────────┘
                          │ 到达清醒时间
                          ▼
                    ┌──────────────┐
              ┌────►│ 🟢 清醒待机  │◄────┐
              │     │  (桌面角落)  │     │
              │     └──────┬───────┘     │
              │            │ 用户点击    │ 对话结束/超时
              │            ▼             │
              │     ┌──────────────┐     │
              │     │ 💬 对话中    │─────┘
              │     │ (人格激活)   │
              │     └──────────────┘
              │
              │     ┌──────────────┐
              │     │ 🎁 每日惊喜  │ ← 每日首次开机触发
              │     └──────────────┘
              │
              └─── 到达睡眠时间 ──→ 🟣 睡眠
```

### 清醒 vs 睡眠 行为对比

| | 🟢 清醒 | 🟣 睡眠 |
|---|---|---|
| LLM 调用 | ✅ 可调用 | ❌ 不调用 |
| 每日惊喜 | ✅ 触发 | ❌ 不触发 |
| 聊天 | ✅ 响应 | ❌ 不响应（或 1 轮迷糊回应） |
| Web Search | ✅ 可搜索 | ❌ 不搜索 |
| 角色动画 | 活跃待机（眨眼/小动作） | 睡眠动画（闭眼/蜷缩/ZZZ） |
| 点击响应 | 正常互动 | "唔……我在睡觉……ZZZ" |
| Token 消耗 | ⚡ 按使用量 | 💤 几乎为零 |

## 核心模块

### 1. 窗口管理器 (Window Manager)

管理桌面角色的**常驻显示**：
- 始终置顶/穿透点击模式（不干扰用户操作）
- 可拖拽到桌面任意位置
- 最小化到系统托盘
- 开机自启注册

### 2. 角色渲染引擎 (Character Renderer)

负责角色的视觉呈现：
- SVG 矢量渲染（轻量、无缩放失真）
- 动画状态机：待机呼吸 → 唤醒 → 说话 → 互动 → 隐藏
- 表情系统：根据人格/内容切换角色表情

**动画状态**：
```
待机 (Idle)
  │  每隔 3-5s 眨眼一次
  │  每隔 10-15s 小动作（耳朵动/尾巴摇/身体晃）
  ▼
唤醒 (Wake) ──→ 出示内容 (Show Content)
  │                    │
  │               用户点击 ──→ 展开详情 (Expand)
  │                    │
  └── 用户忽略 ──→ 退回待机 (Back to Idle)
```

### 3. 人格调度引擎 (Personality Scheduler)

核心逻辑模块：
- 人格包注册/发现（扫描本地人格包目录）
- 人格匹配算法（随机/情境/手动）
- 人格切换（当前人格激活 → 新人格加载 → 过渡动画）
- 人格融合接口（预留）

### 4. 内容获取模块 (Content Fetcher)

负责获取"每日惊喜"的素材：
- **Web Search**：搜索梗图、新闻、热门内容
- **⛅ Weather API**：天气信息（情境触发用）
- **🖼️ 风景 API**：每日一图（Unsplash / Bing）
- **📜 历史 API**：历史上的今天
- **🧠 冷知识库**：结构化冷知识数据集
- **🤖 LLM 补充**：以上都不够时，用 LLM 生成

### 5. 用户画像模块 (User Profile) — Avatara 集成

与 Avatara 系统的双向通道：
- **读取**：用户偏好（兴趣、时间节奏、沟通风格）
- **写入**：用户对内容的反馈（喜欢/不喜欢）
- **权限控制**：用户可随时开关、限制读取范围

### 6. 人格染色器 (Personality Colorizer)

将原始内容转化为人格化表达：
```
输入: { type: "news", content: "Apple 发布新芯片" }
人格: { id: "trump", style: "夸张/咆哮" }
输出: "Apple 新芯片？They say it's the best. 但没有人比我更懂芯片。相信我。"
```

### 7. 睡眠调度器 (Sleep Scheduler)

管理宠物的清醒/睡眠周期：

- 定时器：根据用户配置的清醒/睡眠时间切换状态
- 临时唤醒：睡眠中用户点击 → 允许一次性回应（1 轮对话）
- 对话限额：跟踪每日对话次数，超出后自动进入"节能模式"
- 冷却计时器：对话结束后 30s 冷却
- 与角色渲染引擎联动：状态切换时播放过渡动画（打哈欠 → 入睡 / 伸懒腰 → 醒来）

### 8. 聊天引擎 (Chat Engine)

人格化对话模块：

- **触发检测**：点击/双击/主动搭话 → 判断是否在清醒时段
- **提示词组装器**：动态构建 LLM 请求
  - 当前人格的 voice-profile + worldview + interaction-rules
  - 最近 5 轮对话历史（压缩后）
  - 当前情境上下文（时间、天气、用户状态）
  - Token 预算约束指令
- **回复后处理**：限制输出长度、过滤不安全内容、同步角色动画
- **主动搭话调度**：按配置频率（话痨/正常/高冷/安静）在清醒时段主动问候

### 9. 进化引擎 (Evolution Engine) — 🔮 未来预留

目前只需预留接口和数据存储结构：

```
// MVP 只需存储"互动计数"和"经验值"
{
  "exp": 0,
  "level": 1,
  "evolutionStage": "egg",
  "interactions": 0,
  "daysActive": 0,
  "preferredPersonality": null,
  "unlockedTraits": []
}
```

## 数据存储

| 数据类型 | 存储方式 | 说明 |
|---------|---------|------|
| 人格包 | 文件系统 (JSON + MD) | 每个包一个目录 |
| 用户偏好 | JSON 文件 / SQLite | 睡眠时间、对话配额、主动频率等 |
| 内容缓存 | SQLite / 内存 | 6h 过期 |
| Avatara 数据 | 已有 Avatara 文件 | 只读/追加 |
| 推送历史 | SQLite | 去重 + 回看 |
| 交互反馈 | SQLite | 记录点赞/点踩 |
| 对话历史 | 内存（限 5 轮）+ SQLite 摘要 | 每次会话压缩后存储 |
| 睡眠调度 | 内存（运行期）+ SQLite（配置） | 清醒/睡眠时间配置 |
| 进化数据 | SQLite | EXP、等级、阶段（预留） |

## 跨平台策略

| 平台 | MVP | 正式 |
|------|-----|------|
| macOS | ✅ 目标平台 | ✅ |
| Windows | ✅ 重点覆盖 | ✅ |
| Linux | ⬜ 后续 | ✅ |

> **MVP 建议锁定一个平台**（比如先做 macOS），稳定后再迁移到其他平台。
> Windows 市场份额最大，建议 MVP 优先 Windows。
