# 🥇 MVP 范围与路线图

## MVP 核心必须项（缺一不可）

| # | 能力 | 说明 | 优先级 |
|---|------|------|--------|
| 1 | 🖥️ **桌面常驻组件** | 系统托盘 + 桌面浮动窗口，开机自启，穿透点击模式 | P0 |
| 2 | 🎭 **人格库（≥5 个）** | 内置 5 个以上人格，支持手动切换和随机模式 | P0 |
| 3 | 🌐 **Web Search 集成** | 每日搜索梗图/新闻/热梗，和每日推送直接绑定 | P0 |
| 4 | 🎨 **Iconic 角色形象** | 有辨识度的角色设计 + 至少 3 个动画状态（待机/唤醒/说话） | P0 |
| 5 | 👋 **每日惊喜推送** | 每天首次开机展示一条人格化的内容 | P0 |
| 6 | 💬 **人格化聊天** | 点击角色对话，永远在角色里，≤3 句话/轮 | P0 |
| 7 | 💤 **睡眠/清醒系统** | 配置清醒时段，睡眠时不消耗 token | P0 |
| 8 | 💾 **用户反馈机制** | 点赞/点踩，影响未来推送 | P1 |
| 9 | 🤫 **Avatara 集成** | 读取用户画像影响推送内容，可开关 | P1 |

## 精简 MVP 建议

> 如果追求"一个月内跑起来"，建议精简架构：

```
MVP 技术选型精简方案：
- 前端：Electron + HTML/CSS/JS（最熟的路线，先跑起来）
- 后端：纯 JavaScript（不引入 Python 依赖）
  - Web Search → 内置 fetch API
  - 人格包 → JSON 配置
  - 用户画像 → 本地 JSON 文件
- 角色：Lottie 动画 or 纯 CSS 动画 SVG
```

这套方案一个人能 hold 住，不需要搭建 Python 后端。

## 开发分阶段

### Phase 0：设计与原型（2 周）
- [ ] 角色视觉设计（方向确定 → 概念图 → SVG 定稿）
- [ ] UI/UX 线稿（角色位置、交互流程、人格选择器）
- [ ] 人格包格式规范
- [ ] 技术栈最终确认

### Phase 1：内核（4 周）
- [ ] Electron 框架搭建 + 窗口管理
- [ ] 角色渲染引擎（SVG 动画 + 状态机）
- [ ] 人格调度系统（加载/切换/随机选择）
- [ ] 5 个人格包配置（JSON）
- [ ] 系统托盘 + 开机自启
- [ ] **💤 睡眠/清醒调度器**（定时器 + 状态切换）
- [ ] **💬 基础聊天引擎**（点击触发 + 人格提示词组装 + 短回复约束）

### Phase 2：内容引擎（2 周）
- [ ] Web Search 集成（搜索 + 结果解析）
- [ ] 内容缓存 + 去重
- [ ] 人格染色器（内容 → 人格化表达）
- [ ] 每日惊喜触发器

### Phase 3：用户画像（2 周）
- [ ] 反馈系统（点赞/点踩/跳过）
- [ ] Avatara 数据读取集成
- [ ] 推送权重自适应
- [ ] 设置面板（开启/关闭 Avatara 权限等）

### Phase 4：打磨（1 周）
- [ ] 前后端错误处理与降级
- [ ] 性能优化（内存占用、启动速度）
- [ ] 场景测试（不同网络、不同使用时段的稳定性）

> **总计：约 11 周（~2.7 个月）**

## 验证标准

MVP 完成时，应满足以下标准：

1. ✅ 用户开机 → DesktopCompanion 自动启动
2. ✅ 桌面上出现角色 → 做待机动画 → 弹出今日惊喜
3. ✅ 每日惊喜内容不同 → 且每次有不同人格的"味道"
4. ✅ 用户可以切换到任意人格 → 角色风格随之变化
5. ✅ 内容来源于网上的实时信息（梗图/新闻）
6. ✅ 用户点击角色 → 能对话 → 语气符合当前人格
7. ✅ 对话 ≤3 句话 → 不延伸、不啰嗦
8. ✅ 清醒时段能聊天 → 睡眠时段点击只出睡觉动画 → 不调 LLM
9. ✅ 用户可配置清醒/睡眠时间
10. ✅ 用户点赞/点踩 → 影响后续推荐
11. ✅ 用户不需要打开浏览器就能获得"每天不一样"的体验
