# 📖 产品概念与愿景

## 一句话概括

> **一个懂你、会说话、每天给你惊喜的桌面小生命。**

## 产品愿景

在 AI 时代，桌面辅助工具要么是冷冰冰的效率工具（日历、待办、笔记），要么是过于严肃的 AI 助手（对话窗口、问答机器人）。

**DesktopCompanion 要填补的是"情感陪伴 + 信息价值 + 人格魅力"三者交叉的空白。**

它不是来帮你完成工作的——它是来让 **打开电脑这件事变得有趣** 的。

## 目标用户画像

| 类型 | 痛点 | 我们的解法 |
|------|------|-----------|
| 上班族 | 每天早上不想打开电脑面对工作 | 一个搞笑/温暖的早安，降低心理门槛 |
| 独居工作者 | 缺乏日常人际互动 | 有人格魅力的陪伴感，不是冰冷的提醒 |
| 信息焦虑者 | 想关注新闻但不想刷社交媒体 | 精选推送，不吵不闹 |
| 梗图爱好者 | 每天需要精神食粮 | 今日最佳梗图自动推送 |
| 效率追求者 | 需要早间信息简报 | 新闻搜集小能手模式 |

## 核心体验：每日"惊喜"机制

每日首次唤醒（开机/解锁）时触发：

```
[场景] 用户打开电脑
[触发] DesktopCompanion 启动
[决策] 人格库 × 每日内容 × 情境 → 生成今日惊喜
[呈现] Iconic 角色出现 + 一条个性化消息
[后续] 用户点击/交互 → 展开更多
```

惊喜类型（每日随机一种）：

| 类型 | 示例 | 数据来源 |
|------|------|---------|
| 🖼️ 风景大图 | "今天带你去冰岛看极光 🌌" | Unsplash / Bing Wallpaper API |
| 🤡 搞笑梗图 | 一张猫 meme | Web Search / 梗图源 |
| 📜 历史上的今天 | "1980 年的今天，微软的 MS-DOS 发布了……" | Wikipedia / 历史 API |
| 🎭 人格模仿 | "I'm back, baby. Make America Great Again!" — 特朗普 | Nuwa 人格库 |
| 📰 今日必读 | "某明星大瓜！" / "今日科技头条 3 条" | Web Search / RSS |
| 🧠 冷知识 | "你知道吗？章鱼有三个心脏。" | 冷知识库 / LLM 生成 |
| ✨ 定制化 | 根据 Avatara 对用户的了解推荐内容 | Avatara 用户画像 |

## 💤 睡眠/清醒机制 — Token 经济核心

> **"不是冷血，是会过日子。"**

为了在"陪伴感"和"token 成本"之间取得平衡，DesktopCompanion 采用**睡眠/清醒双状态**设计。

### 状态定义

| 状态 | 行为 | Token 消耗 | 视觉表现 |
|------|------|-----------|---------|
| 🟢 **清醒** | 可聊天、可推送、可互动 | ⚡ 有消耗（LLM 调用） | 眼睛睁开，活跃待机动画 |
| 🟣 **睡眠** | 不说话、不推送、不响应对话 | 💤 几乎为零 | 闭眼/蜷缩/ZZZ 动画 |

### 清醒时间配置

清醒时间由用户自定义（默认推荐）：

```
推荐默认配置：
🕐 清醒时段：08:00 — 22:00（共 14 小时）
🕐 睡眠时段：22:00 — 08:00（共 10 小时）

配置项：
- 清醒开始时间（默认 08:00）
- 睡眠开始时间（默认 22:00）
- 是否允许临时唤醒（默认：允许）
```

### 临时唤醒机制

即使处于睡眠时段，用户点击睡着的宠物 → 它迷糊地醒来：
- 只回应 **1 轮对话**（说完继续睡）
- 或者配置 "睡眠中不打扰" 模式（完全不响应）
- 示例：半夜点击它 → 睡眼惺忪："唔……几点了……明天还要上班呢……ZZZ"

### 清醒时的 Token 管理

| 机制 | 说明 |
|------|------|
| **每日对话限额** | 清醒时段内每天 N 次免费对话，超出后提示"有点累了" |
| **主动 vs 被动** | 宠物主动推送（每日惊喜）不计入对话限额 |
| **冷却时间** | 同一轮对话结束后，30 秒冷却才能再次唤起（防刷） |
| **敏感时段静音** | 用户配置"工作中勿扰"时段，宠物保持安静 |

## 💬 聊天能力

### 触发方式

| 方式 | 说明 |
|------|------|
| 👆 **点击角色** | 宠物被戳一下 → "嗯？怎么啦？" |
| 🖱️ **双击/长按** | 进入聊天模式 → 弹出对话气泡 |
| ⌨️ **自动搭话** | 清醒时段偶尔主动问一句（频率可配置：高/中/低/不主动） |

### 聊天原则

- **永远在角色里** — 不管聊什么，语气和世界观与当前人格一致
- **简短为美** — 默认单次回复 ≤ 3 句话。如果用户追问才展开
- **不乱延伸** — 不主动开启无关话题，不"越聊越嗨"
- **可随时终止** — 用户 ESC 或点击桌面空白处 → 宠物自动闭嘴退回收纳状态

### 聊天示例

```
[当前人格：傲娇猫]
用户点击 → 猫（眼神移开）："哼？干嘛啦……"
用户："今天好累"
猫（耳朵动了动）："……那、那你休息一下也不是不行。我又不是在关心你。"

[当前人格：特朗普]
用户点击 → 川普："What's up! 你来了！"
用户："今天有什么新闻？"
川普："Let me tell you——今天的新闻，huge! 但是你听了别哭，OK？"
```

## 进化系统（未来构想 🧪）

> *以下不是 MVP 内容，是未来方向的灵感种子。*

受电子宠物（拓麻歌子）启发，未来可能引入**成长进化**机制：

```
🥚 蛋 → 🐣 幼体 → 🐾 成体 → 🌟 完全体 → ？？？
         ↓          ↓          ↓
     每天浇水     持续互动   人格觉醒
     （登录）    （对话+反馈） （拥有自己的"芯"）
```

### 进化驱动因素

| 行为 | 影响 |
|------|------|
| 每天开机登录 | 经验值 +1，维持生命值 |
| 与宠物对话 | 经验值 +1~3（依对话深度） |
| 使用宠物推送的内容 | 经验值 +1 |
| 长期不理它 | 生命值下降，角色呈现"萎靡"状态 |
| 给它换人格 | 解锁进化分支的触发条件 |

### "拥有自己的人格"

这是最有趣的部分——经过长期互动后，宠物可能**不再只是模仿别人**，而是基于：
- 用户最常使用的人格组合
- 用户对各类内容的反馈偏好
- 互动历史中的模式
- 进化阶段

→ **融合出它自己独一无二的"芯" personality**

> 这个方向如果真的做出来，就是真正的差异化——别人做的是人格收集，我们做的是 **人格孕育**。

## 品牌调性

- **不严肃** — 它是个宠物，不是助理。叫它"助手"它会不高兴
- **有惊喜** — 每天不一样是硬性要求，不是可选项
- **有灵魂** — 人格库不是噱头，是核心竞争力
- **不打扰** — 消息在桌面常驻，但不弹窗轰炸。用户忙的时候它会安静呆着
- **会过日子** — 该睡睡该醒醒，不浪费你的 token
- **会长大** —（未来）你养它，它也在成长

## 长期愿景

```
阶段一：单人陪伴        → 阶段二：社交互动        → 阶段三：生态平台
一个懂你的桌面宠物      宠物之间可以串门/聊天      人格市集，人人可创造
       │                        │
       └── 进化系统             └── 进化系统
       （养大你的宠物）         （宠物社交 + 配种进化）
```

但在此之前，先把 MVP 做扎实。
