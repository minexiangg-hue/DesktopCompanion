# 🎭 人格库系统设计

> **核心竞争力所在。** 别的桌宠卖萌，我们能卖人格。

## 系统概述

人格库（Personality Library）是 DesktopCompanion 的灵魂引擎。它负责：
- 管理所有可用人格的注册、加载、切换
- 根据情境/随机/手动三种模式调度人格
- 将人格特质注入每日消息生成

## 人格来源

### 主要来源：Nuwa 蒸馏人格

通过 Nuwa Skill 蒸馏的每一个名人/角色人格，都产出为标准化的「人格包」(Personality Pack)：

```
personality-pack/
├── manifest.json        # 元数据（名称、标签、风格分类）
├── voice-profile.md     # 语气、口癖、句式特征
├── worldview.md         # 核心信念、价值观
├── knowledge-base.md    # 该人格擅长的知识领域
└── interaction-rules.md # 什么语境下该说什么话
```

### 人格分类

| 分类 | 示例 | 适合场景 |
|------|------|---------|
| 🎬 名人 | 特朗普、马斯克、罗翔、鲁迅 | 早起震撼/毒舌/金句 |

## 人格调度系统

### 三种调度模式（并存）

#### 1. 🎲 随机模式 — "今天翻谁牌子？"

```
每天首次启动时 → 从人格库中随机选择一人格
筛选条件：排除昨天已用的 / 根据用户活跃时段匹配
权重：新人格权重稍高（探索），已用过的人格衰减（利用）
```

**算法**：加权随机 + 探索率 ε-greedy
- ε = 0.3：30% 概率完全随机探索新人格
- 1 - ε = 0.7：70% 概率从"用户反馈好"的人格中选

#### 2. 🎯 情境触发模式 — "你该被骂醒了"

通过系统信号检测情境，自动匹配最合适的人格：

| 系统信号 | 检测方式 | 推荐人格 | 示例话术 |
|---------|---------|---------|---------|
| 凌晨 2 点还在亮屏 | 系统运行时间 > 16h | 老妈人格 | "几点了还在熬？不要命了？" |
| 周一早上 8-10 点 | 星期检测 | 死侍人格 | "哦豁，又是周一。你老板给你涨工资了吗？" |
| 用户长时间摸鱼（无操作 > 10min） | 空闲检测 | 傲娇猫 | "哼，终于想起我了？" |
| 天气恶劣 | 天气 API | 温暖型人格 | "今天下雨，带伞了吗？" |
| 连续工作 2h 以上 | 活跃度检测 | 搞笑人格 | "歇会儿吧哥，你这么拼让我很焦虑" |
| 新项目/任务创建 | 系统通知 | 导师型人格 | "有意思，说说你的计划？" |

#### 3. 👆 手动切换 — "我自己翻牌"

用户从桌面右键/点击角色 → 人格选择器：
- 网格展示人格库（带头像/颜色/一句简介）
- 可收藏/置顶常用人格
- 最近使用记录

### 人格融合（进阶）

两到三个人格可以"融合"生成混合人格：
- 特朗普 × 鲁迅 = 用咆哮体讲犀利道理
- 福尔摩斯 × 傲娇猫 = 高冷理性但偶尔撒娇

> **MVP 暂不实现**，但架构上预留接口。

## 人格与每日惊喜的耦合

每日惊喜不是"人格说一句话"，而是：
1. 选定人格 → 选定内容类型（梗图/新闻/风景…）
2. 人格调性决定**呈现方式**：

| 内容类型 | 特朗普 | 鲁迅 | Chiikawa 风萌宠 |
|---------|--------|------|----------------|
| 风景大图 | "This is the most beautiful picture you've ever seen. Believe me." | "向来如此，便对么？——不过风景倒是好风景" | "哇——！好漂酿！🌟" |
| 梗图 | "Sad! Low energy!" | "人类的悲欢并不相通，但好笑是相通的" | "噗哈哈哈哈笑死了🤣" |
| 新闻 | "Fake news! But this one... this one is huge." | "我向来不惮以最坏的恶意揣测……嗯这条还行" | "诶诶诶！大新闻！！" |
| 冷知识 | "Nobody knows more about octopuses than me." | "读书人的事，能算冷吗？" | "哇塞！章鱼有3个心脏？！" |

## 💬 人格化聊天系统

> 不管聊什么——永远在角色里。

### 聊天触发条件

聊天仅在**清醒时段**可用。睡眠时段点击无效（或只触发睡梦呓语）。

| 触发方式 | 说明 |
|---------|------|
| 点击角色 | 简短问候，1-2 句交互 |
| 双击/长按 | 进入深度聊天模式，弹出对话窗口 |
| 主动搭话 | 清醒时段按配置频率主动问候 |

### 人格驱动的对话策略

每个人格包需定义以下聊天行为规范：

```json
{
  "chatBehavior": {
    "greetingChance": 0.3,
    "greetings": ["嗯？", "干嘛？", "你来了。", "Hi!"],
    "maxLinesPerTurn": 3,
    "maxTurnsBeforeClosure": 5,
    "topics": {
      "preferred": ["工作", "美食", "吐槽"],
      "avoid": ["政治敏感", "负面情感"],
      "defaultTo": "日常闲聊"
    },
    "closureStyle": "abrupt | gentle | funny",
    "canAskQuestions": true,
    "questionFrequency": "low | medium | high"
  }
}
```

### 对话回复管线

```
用户输入
  │
  ├─ 人格包 chatBehavior 加载
  │
  ├─ 系统提示词组装：
  │   当前人格 voice-profile + worldview + interaction-rules
  │   + 压缩的对话历史（最多 5 轮）
  │   + 用户画像上下文（如可用）
  │   + Token 预算指令（≤3 句话，不主动延伸）
  │
  ├─ LLM 调用（本地模型 or API）
  │
  └─ 输出 → 角色动画同步 → 对话气泡呈现
```

### Token 优化策略

| 优化手段 | 说明 |
|---------|------|
| **短回复约束** | system prompt 强制 ≤3 句话，控制输出 token |
| **对话轮次上限** | 单次聊天会话不超过 5 轮，超时自动结束 |
| **冷却时间** | 每轮对话后 30s 冷却 |
| **上下文压缩** | 只保留最近 3-5 轮对话，不累积膨胀 |
| **本地模型备选** | 简单寒暄走本地小模型，深度对话走 API |
| **对话限额** | 每日免费 N 次（可配置），超出后宠物说"今天有点累了" |

### 主动搭话策略（清醒时段）

宠物可以主动和用户说话，但频率可控：

| 模式 | 行为 | Token 影响 |
|------|------|-----------|
| 🐣 话痨 | 每小时主动问候一次 | 较高 |
| 🐱 正常 | 每天 3-5 次主动（推荐） | 中等 |
| 🐻 高冷 | 只在每日惊喜时说话 | 低 |
| 🦉 完全安静 | 只有用户点击才回应 | 最低 |

## 人格库的扩展
