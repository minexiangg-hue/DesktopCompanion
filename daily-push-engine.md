# 📰 每日推送引擎 + Web Search 集成

## 系统定位

每日推送引擎（Daily Push Engine）是 DesktopCompanion 的"内容大脑"。它决定 **今天给用户看什么**。

## 架构概览

```
[时序触发]                    [内容源]                    [输出]
每天首次开机 ──→ 人格调度 ──→ ├─ Web Search（梗图/新闻）
                              ├─ 风景 API（每日一图）
                              ├─ 历史 API（历史上的今天）
                              ├─ 冷知识库
                              └─ Avatara 画像（个性化）
                                    │
                                    ▼
                              内容选择器（上下文感知）
                                    │
                              ┌────┴────┐
                              │ 人格染色  │ ← 人格库决定"怎么说"
                              └─────────┘
                                    │
                                    ▼
                              桌面角色呈现
```

## Web Search 集成

### 主要搜索主题

| 搜索主题 | 查询策略 | 刷新频率 | 用途 |
|---------|---------|---------|------|
| 🔥 今日热梗 | "today's best memes" / "今日梗图 top" | 每日 1 次 | 搞笑推送 |
| 📰 头条新闻 | "今日热搜" / "trending news today" | 每日 2 次（早/晚） | 新闻推送 |
| 💼 行业动态 | 根据用户画像定向搜索 | 每日 1 次 | 工作相关内容 |
| 🎉 娱乐八卦 | "celebrity news today" | 每日 1 次 | 吃瓜推送 |
| 📅 历史今日 | "On this day history" | 每日 1 次 | 知识推送 |

### 内容筛选原则

| 原则 | 说明 |
|------|------|
| ✅ 信息增益 | 内容要有"哇"因子，垃圾信息过滤 |
| ✅ 情绪安全 | 不推送负面新闻（灾难/凶杀/政治敏感） |
| ✅ 来源可信 | 优先来自主流媒体/权威来源 |
| ❌ 重复内容 | 同一条信息不推两次（去重缓存） |
| ❌ 用户不感兴趣 | Avatara 学习用户偏好后自动过滤 |

## 每日惊喜的生成管线

### Step 1：触发时机（每日首次）

```
触发条件：
- 系统开机 / 用户解锁屏幕 / 从睡眠恢复
- 且距离上次推送 > 12 小时（防止一天推多次）
```

### Step 2：人格筛选

从人格库中按当前模式（随机/情境/手动）选择一个人格。

### Step 3：内容类型选择

```
随机权重分配（MVP 默认，后期可由用户调节）：
- 搞笑梗图：25%
- 风景大图：15%
- 历史上的今天：15%
- 人格经典语录：15%
- 今日新闻/热梗：20%
- 冷知识：10%
```

### Step 4：内容获取

并行请求多个内容源，取最快返回的结果（保证推送速度）：

```
1. Web Search → 爬取/解析 → 内容提取
2. API 调用 → 风景 / 新闻 / 历史
3. 本地生成 → 冷知识 / 人格语录
```

### Step 5：人格染色

将获取的原始内容，通过当前人格的 voice-profile 重新表达：

```
原始内容："今天是 2026 年 6 月 7 日，程序员节。"
[特朗普人格] → "程序员节？Nobody loves programmers more than me. 相信我，我们是最好的程序员。"
[鲁迅人格]   → "世界上本没有节日，过的人多了，也便成了节。"
[萌宠人格]   → "程序员节！！！大佬们要开心敲代码哦！💻✨"
```

### Step 6：桌面呈现

角色出现 → 推送消息 → 根据用户交互决定后续：

```
用户点击角色 → 展开详细内容（梗图大图 / 新闻全文 / 风景壁纸）
用户忽略     → 30 秒后角色退到桌面角落待机
用户关闭     → 今天不再显示（但保留在托盘区）
用户点赞 👍  → 记录偏好，影响未来推送权重
用户点踩 👎  → 此类内容降权
```

## 技术要点

### 搜索内容缓存

```
缓存策略：
- 搜索结果缓存 6 小时（同一天内容不变）
- API 结果缓存 24 小时（风景/历史今日不变）
- 用户交互反馈实时记录
```

### 去重机制

```
Bloom Filter（搜索内容哈希） + Redis Set（已推内容 ID）
确保同一张梗图/同一条新闻不会出现两次
```

### Web Search 失败时的降级

```
搜索超时/失败 → 一降：使用缓存内容 → 二降：使用纯人格语录 → 三降：角色卖个萌
"网络好像不太给力呢… 那我先陪你聊聊天吧！"
```

## Avatara 集成：个性化推送

一旦用户授权 Avatara 读取其聊天记录/文件，推送引擎将：

1. **学习兴趣领域** — 用户常聊/关注的话题优先搜索
2. **学习时间偏好** — 用户早上看新闻还是看梗图？自动调节权重
3. **学习禁入话题** — 用户从不聊/反感的话题自动屏蔽
4. **学习沟通风格偏好** — 喜欢精炼？详细？诙谐？严肃？
